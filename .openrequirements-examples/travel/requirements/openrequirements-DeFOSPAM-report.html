📚 Proposed Glossary

Key terms identified in requirements analysis with their proposed definitions and verification status.

Disruption Detection
The process of identifying and ingesting adverse events affecting transportation services from operational data sources
✓ Verified
Correlation
The process of matching a normalized disruption event to a passenger booking using deterministic matching strategies
✓ Verified
Case
A distinct record representing a specific passenger journey affected by a disruption event, created upon successful event correlation
✓ Verified
Idempotency Key
A unique identifier combining Case, channel, and template that ensures the same notification is never sent twice
✓ Verified
Exactly One Compliant Passenger Notification
A requirement ensuring that for a specific Case, channel, and template, exactly one outbound message is delivered

Lead Contact
The passenger designated as the primary contact for a booking reference and recipient of disruption notifications
✓ Verified
Consent State

✓ Verified
Case Timeline
